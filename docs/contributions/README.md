# Contributions Tracking

This directory tracks all contributions (PRs, issues, discussions) to upstream projects.

## Format

Each contribution gets a markdown file:

```
pr-001-feature-name.md
issue-042-bug-description.md
discussion-015-topic.md
```

## Template

```markdown
# [PR|Issue|Discussion] #NNN: Title

**Upstream Repo**: danielmiessler/PAI
**Status**: Draft | Open | Merged | Closed
**Created**: 2025-12-02
**Link**: https://github.com/...

## Summary
Brief description of the contribution

## Motivation
Why this change is valuable

## Implementation
Technical approach taken

## Discussion Points
Key feedback from maintainers

## Outcome
What happened (merged, closed, modified, etc.)

## Learnings
What I learned from this contribution
```

## Contribution Goals

- [ ] First PR submitted
- [ ] First PR merged
- [ ] Bug fix contributed
- [ ] Feature contributed
- [ ] Documentation improvement
- [ ] Helped another contributor

## Guidelines

1. **Quality over quantity** - Focus on valuable contributions
2. **Respect maintainer time** - Well-documented, tested PRs only
3. **Accept feedback gracefully** - It's a learning process
4. **Follow project conventions** - Match their style and approach
5. **Be patient** - OSS maintainers are often volunteers

---

*Remember: Every contribution is a learning opportunity, whether merged or not*
